## Faz 7: Bağlantılar ve Tetiklemeler (Topic / Flow) — Süre: ~45 dakika

Bu son fazda tüm parçaları birbirine bağlıyoruz. Önce Faz 6'te oluşturduğumuz hesaplama flow'unu
`Anket ve Hesaplama` topic'ine bir araç olarak ekliyoruz; ardından hesaplama sonuçlarını
kullanıcıya hem ham hem de yorumlanmış biçimde sunuyoruz. Son adımda ise ana topic'in iki farklı
dalını `Anket ve Hesaplama` topic'ine yönlendirerek kapatıyoruz.

Bu faz sonunda agent; tüm fazlardaki akışı eksiksiz çalıştırabilecek, anket sonuçlarını
Excel üzerinden matris hesaplamasına sokabilecek ve kullanıcıya hem kriter ağırlıklarını hem de
yapay zeka destekli bir karar özeti sunabilecek tam kapasiteye ulaşacaktır.

### Challenge 7.1: Flow'u Topic'e Araç Olarak Bağlama

**Hedef:** Faz 6'te oluşturulan hesaplama flow'unu `Anket ve Hesaplama` topic'ine eklemek,
input ve output değişkenlerini bağlamak.

**Adımlar:**

1. Faz 6'te yer alan flow yapısını oluşturmaya agent sekmesinden başladığınız, akış otomatik olarak agent sekmesinde görülebilecektir. Eğer görünmüyorsa 1.1 adımını takip edin, görünüyorsa 2.adıma geçebilirsiniz.

1.1 `Anket ve Hesaplama` topic'ini açın (Faz 5'ün son node'unun
devamına ekleme yapacaksınız).

1.2 En alta inin ve **+** → **Add a tool** seçin. Listede Faz 6'te oluşturduğunuz flow
kendiliğinden görünebilir; görünmüyorsa listeyi yenileyerek ilgili flow'u seçin.
Seçim yapıldığında canvas'a bir **Action** node'u eklenir.

2.Action node'unun **Input** bölümünde **⋯ (üç nokta)** butonuna tıklayın.
Açılan panelden flow'a gönderilecek altı değişkeni sırayla bağlayın:

| Flow Input | Bağlanacak Değişken |
| ---------- | ------------------- |
| `q1winner` | `q1winner`          |
| `q2winner` | `q2winner`          |
| `q3winner` | `q3winner`          |
| `q1score`  | `q1score`           |
| `q2score`  | `q2score`           |
| `q3score`  | `q3score`           |

3.**Output** bölümünde **⋯** → **Create a variable** ile altı yeni global değişken oluşturun:

| Flow Output        | Yeni Değişken Adı |
| ------------------ | ----------------- |
| `Criteria Results` | `CriteriaResult`  |
| `Consistency`      | `Consistency`     |
| `Top Criteria`     | `TopCriteria`     |
| `Supplier Scores`  | `SupplierScores`  |
| `Best Supplier`    | `BestSupplier`    |
| `Best Score`       | `BestScore`       |

**✅ Beklenen Çıktı:**

- Action node'unda altı input ve altı output bağlıdır.

### Challenge 7.2: Hesaplama Özeti Üretme ve Topic'i Kapatma

**Hedef:** Hesaplama çıktılarını yapay zekayla yorumlatmak, özeti kullanıcıya sunmak ve
topic'i düzgün biçimde sonlandırmak.

**Adımlar:**

1. **+** → **add a tool** → **new prompt** ile Create a prompt node'u ekleyin. **Model** olarak **GPT-5 Chat** seçin.

2. Prompt metnine aşağıdakileri yapıştırın. Metni kaydedin, ardından her `{değişken}` yer
   tutucusu için **Add content** → **Text** türünde input oluşturun ve **Name** alanını
   tablodaki isme göre değiştirin. Tüm input'lar eklendikten sonra tekrar **Save** edin:

   ```text
   Aşağıdaki AHP değerlendirme çıktısını analiz et ve kısa, profesyonel,
   iş odaklı bir özet üret.

   Bağlam:
   Bu değerlendirme satın alma karar desteği amacıyla yapılmıştır.

   Kriter yapısı:
   {criteriafinal}

   Kullanıcı karşılaştırma yanıtları:
   - Soru 1 sonucu: {q1winner}, skor: {q1score}
   - Soru 2 sonucu: {q2winner}, skor: {q2score}
   - Soru 3 sonucu: {q3winner}, skor: {q3score}

   Hesaplanan kriter ağırlıkları:
   {criteriaresult}

   Tutarlılık oranı:
   {consistency}

   En güçlü kriter:
   {topcriteria}

   Görev:
   Aşağıdaki başlıklarda kısa bir değerlendirme özeti üret:
   1. Karşılaştırma Özeti
   2. Önceliklendirme Yorumu
   3. Tutarlılık Değerlendirmesi
   4. Karar Destek Notu

   Kurallar:
   - Açık, profesyonel ve kısa yaz.
   - Teknik formül açıklama.
   - Yalnızca verilen verilere dayan.
   - Gereksiz tekrar yapma.
   - Sonuç odaklı yaz.
   ```

3. **Input** bölümünde her yer tutucu için mevcut değişkeni seçin (yeni değişken
   oluşturulmaz):

   | Prompt Input     | Bağlanacak Değişken |
   | ---------------- | ------------------- |
   | `criteriafinal`  | `CriteriaFinal`     |
   | `q1winner`       | `q1winner`          |
   | `q1score`        | `q1score`           |
   | `q2winner`       | `q2winner`          |
   | `q2score`        | `q2score`           |
   | `q3winner`       | `q3winner`          |
   | `q3score`        | `q3score`           |
   | `criteriaresult` | `CriteriaResult`    |
   | `consistency`    | `Consistency`       |
   | `topcriteria`    | `TopCriteria`       |

4. **Output** bölümünde yeni global bir değişken oluşturun:
   - Ad: `AHPSummary`

5. **Save** butonuna tıklayın.

6. Action node'u canvas'a geldiğinde tüm input değişkenlerinin listeden seçili olduğunu
   ve output'un `AHPSummary` olarak atandığını doğrulayın.

7. **+** → **Send a message** ekleyin:

   ```text
   AHP süreç özeti:
   ```

   Ardından **{x} Insert variable** → `AHPSummary.text` seçin.

8. **+** → **Send a message** ekleyin. Aşağıdaki mesajı oluşturun; her satırdaki değişken
   için **{x} Insert variable** ile ilgili değişkeni seçin:

   ```text
   AHP Değerlendirme Sonucu

   Öne Çıkan Kriter:
   ```

   {x} → `TopCriteria`

   ```text
   Tutarlılık Oranı:
   ```

   {x} → `Consistency`

   ```text
   Tedarikçi Karşılaştırma Skorları:
   ```

   {x} → `SupplierScores`

   ```text
   En Uygun Tedarikçi:
   ```

   {x} → `BestSupplier`

   ```text
   En Uygun Tedarikçi Skoru:
   ```

   {x} → `BestScore`

9. **+** → **Topic management** → **End current topic** seçin.

10. **Save** butonuna tıklayın.

**✅ Beklenen Çıktı:**

- Create a prompt node'unda on input bağlıdır; output `AHPSummary`'dir.
- İlk Send a message node'unda `AHPSummary.text` mesaja eklenmiştir.
- İkinci Send a message node'unda tedarikçi sonuçları (`TopCriteria`, `Consistency`,
  `SupplierScores`, `BestSupplier`, `BestScore`) eklenmiştir.
- Canvas'ın sonunda End current topic node'u bulunmaktadır.

### Challenge 7.3: Ana Topic'ten Yönlendirmeleri Bağlama

**Hedef:** Ana topic'teki iki farklı onay dalını `Anket ve Hesaplama` topic'ine
yönlendirerek tam akışı kapatmak.

**Adımlar:**

1. Faz 3'de oluşturulan ana topic'e geçin (şartname analizi akışının bulunduğu topic).

2. Canvas'ta koşul bloğunu bulun. **"Uygun, devam et"** dalındaki `Set variable`
   (`CriteriaFinal = CriteriaDraft`) node'unun altına inin.

3. **+** → **Topic management** → **Go to another topic** seçin.
   Açılan topic listesinden `Anket ve Hesaplama`'yı seçin.

   > Seçim yapıldığında bu dalda artık anket topic'ine yönlendirme tanımlıdır.
   > Kullanıcı "Uygun, devam et" seçtiğinde doğrudan anket ve hesaplama akışına geçilir.

4. **+** → **Topic management** → **End all topics** seçin.

5. **"Düzenlemek istiyorum"** dalına geçin. Faz 4'te eklenen son node'a (Set variable:
   `CriteriaFinal = RevisedCriteriaFinal`) inin.

6. Aynı adımları tekrarlayın: **+** → **Topic management** → **Go to another topic** →
   `Anket ve Hesaplama`.

7. **+** → **Topic management** → **End all topics** seçin.

8. **Save** butonuna tıklayın.

9. Bu adımdan sonra agent ile konuşmanın sonlanması ve sürecin tamamlanması için,
   **+** → **Topic management** → **End conversation** seçin.

**✅ Beklenen Çıktı:**

- Ana topic'in "Uygun, devam et" dalı `Anket ve Hesaplama`'ya yönlendirilmiştir.
- "Düzenlemek istiyorum" / revizyon dalı da aynı topic'e bağlanmıştır.
- Test panelinden akışın başından sonuna kadar kesintisiz çalıştığını doğrulayın:
  belge yükleme → özetleme → kriter belirleme → onay/revizyon → anket → matris hesaplama →
  özet görüntüleme.
